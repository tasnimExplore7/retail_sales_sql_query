# retail_sales_sql_query 

# Retail Sales Analysis Using SQL

## Project Overview

This project analyzes retail sales data using PostgreSQL. The goal is to perform data cleaning, data exploration, and business analysis to generate meaningful insights from customer transactions.

The project covers the complete analytics workflow, including handling missing values, validating data quality, and answering business questions through SQL queries.

---

## Dataset Information

The dataset contains retail transaction records with the following fields:

| Column Name     | Description                   |
| --------------- | ----------------------------- |
| transactions_id | Unique transaction identifier |
| sale_date       | Date of sale                  |
| sale_time       | Time of sale                  |
| customer_id     | Unique customer identifier    |
| gender          | Customer gender               |
| age             | Customer age                  |
| category        | Product category              |
| quantiy         | Quantity purchased            |
| price_per_unit  | Unit price of product         |
| cogs            | Cost of goods sold            |
| total_sale      | Total sales amount            |

---

## Data Cleaning

### Null Value Detection

Checked all columns for missing values using SQL queries.

### Missing Age Handling

Instead of removing records immediately, missing values in the `age` column were replaced with the average customer age.

### Record Removal

Remaining records containing null values in critical columns were removed to maintain data quality.

## Business Questions Solved

### 1. Sales on a Specific Date

Retrieved all transactions made on a selected date.

### 2. Clothing Sales Analysis

Identified clothing transactions and analyzed purchased quantities during November 2022.

### 3. Category Performance

Calculated total revenue generated by each product category.

### 4. Customer Age Analysis

Determined the average age of customers purchasing Beauty products.

### 5. High-Value Transactions

Identified transactions where total sales exceeded 1000.

### 6. Gender-Based Purchase Analysis

Analyzed transaction counts by gender across different categories.

### 7. Monthly Sales Trends

Calculated average monthly sales and identified the best-performing month in each year using window functions and ranking.

### 8. Top Customers

Identified customers with the highest total spending.

### 9. Category Customer Distribution

Calculated the number of unique customers purchasing from each category.

### 10. Sales Shift Analysis

Categorized transactions into:

* Morning (< 12 PM)
* Afternoon (12 PM – 5 PM)
* Evening (> 5 PM)

and analyzed order volume for each shift.

## Key Insights

* Customer purchasing behavior varies across product categories.
* Certain months consistently generate higher average sales.
* A small number of customers contribute significantly to overall revenue.
* Sales activity differs throughout the day, with specific shifts generating more orders.
* Product category performance can be measured effectively through revenue and customer engagement metrics.
